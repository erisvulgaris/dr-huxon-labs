   132→   132→- Created `src/lib/appwrite/server.ts`:
   133→   133→  - Reads `APPWRITE_ENDPOINT` (default `http://localhost:8080/v1`), `APPWRITE_PROJECT_ID`, `APPWRITE_API_KEY` from env.
   134→   134→  - Lazy singleton `Client` + `Databases` instances (safe to import even when unused).
   135→   135→  - Exported constants: `HUXON_DB_ID = "huxon_labs"` + `PRODUCTS_COLLECTION`, `INGREDIENTS_COLLECTION`, `REVIEWS_COLLECTION`, `COUPONS_COLLECTION`, `ORDERS_COLLECTION`, `REWARD_MEMBERS_COLLECTION`.
   136→   136→  - `isAppwriteConfigured()` checks env presence; `isAppwriteReachable()` does a live `listDocuments(limit:1)` ping.
   137→   137→- Created `src/lib/appwrite/service.ts` — the KEY resilient data layer:
   138→   138→  - Public typed async functions: `fetchProducts()`, `fetchProduct(slug)`, `fetchIngredients()`, `fetchReviews(productId?)`, `fetchCoupons()`, `validateCoupon(code, subtotal)`, `submitOrder(payload)`, `fetchRewardMember(email)`.
   139→   139→  - Each read method: tries Appwrite when configured + reachable, falls back to `src/lib/catalog.ts` static data on ANY error (offline, auth, schema drift, empty result). Connection result is cached per-process to avoid a round-trip on every request.
   140→   140→  - Mutations (`submitOrder`) always return an `orderNumber` even if persistence fails — checkout never blocks on infra.
   141→   141→  - Document → typed mappers (`docToProduct`, `docToIngredient`, `docToReview`, `docToCoupon`, `docToRewardMember`) with JSON-aware string parsing for `features`, `ingredients`, `nutritionFacts`, `pairings`, `galleryImages`, `benefits`, `achievements`.
   142→   142→  - Local fallback data for coupons (`HUXON10`, `WELCOME500`, `PLANT15`) and a sample reward member (`arjun@example.com`).
   143→   143→- Created 8 Next.js 16 App-Router API routes (all `dynamic = "force-dynamic"`, all return JSON):
   144→   144→  - `GET /api/products` → `{ products: BrandProduct[] }`
   145→   145→  - `GET /api/products/[slug]` → `{ product: BrandProduct }` (404 if missing)
   146→   146→  - `GET /api/ingredients` → `{ ingredients: BrandIngredient[] }`
   147→   147→  - `GET /api/reviews?productId=p1` → `{ reviews: Review[] }`
   148→   148→  - `GET /api/coupons` → `{ coupons: Coupon[] }`
   149→   149→  - `POST /api/coupons/validate` body `{code, subtotal}` → `{valid, discount, message}` (400 on bad input, 200 otherwise)
   150→   150→  - `POST /api/orders` body = `OrderPayload` → `{orderId, orderNumber, status}` (201 on success, 400 on validation errors)
   151→   151→  - `GET /api/rewards/member?email=` → `{ member: RewardMember }` (404 if unknown)
   152→   152→- Created `scripts/setup-appwrite.ts` (runnable via `bun scripts/setup-appwrite.ts`):
   153→   153→  - Creates the `huxon_labs` database and 6 collections (Products, Ingredients, Reviews, Coupons, Orders, Reward Members) with full attribute schemas + indexes (`slug_idx`, `product_idx`, `code_idx`, `order_number_idx`, unique `email_idx`).
   154→   154→  - Polls `getAttribute` until each attribute is `available` before continuing (Appwrite attribute creation is async).
   155→   155→  - Seeds products (6), ingredients (6), reviews (4), and 3 coupons from `catalog.ts` when collections are empty (idempotent — safe to re-run).
   156→   156→  - Graceful exit 0 in all "can't proceed" cases: env vars missing → friendly skip message; server unreachable → friendly "Could not reach Appwrite" message; any unexpected error → caught at top level, exit 0.
   157→   157→- Updated `.env` (added 3 empty Appwrite vars alongside existing `DATABASE_URL`) and created `.env.example` with full documentation of what each var does and how to enable a live backend.
